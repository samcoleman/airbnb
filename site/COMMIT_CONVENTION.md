# Commit Message Convention

All commits in this project should follow the conventional commit format with the `site` scope:

```
type(scope): description

Examples:
- feat(site): add new feature
- fix(site): fix bug
- refactor(site): restructure code
- docs(site): update documentation
- style(site): improve formatting
- test(site): add tests
- chore(site): maintenance tasks
```

The `site` scope should be used consistently for all commits in the `/site` directory.
